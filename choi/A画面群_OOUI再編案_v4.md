# A画面群 OOUI再編案 v4

## リリース計画とドッグフーディング

**フェーズ1(本設計スコープ)**: トピックメーカー内ドッグフーディング運用
- `isトピックメーカー=true` の団体(=株式会社トピックメーカー)の
  社員が投稿システムを使用する
- ただしトピックメーカー社員が**様々なロールでドッグフーディング**することで
  フェーズ2で想定される全ユーザーの体験を事前検証する
  - 団体管理者として使う
  - 団体一般会員として使う
  - 無所属会員として使う(社員が一時的に無所属状態をテスト)
- したがって**全ロール向けの画面を本フェーズで実装する**
- アクセスはトピックメーカー専用暫定プロフィール画面経由
  (社員のみが知るURL)

**フェーズ2以降**: 一般公開
- 正規のプロフィール画面に統合、アクセス制御も正式化

## システム構造(v2から継承)

**コース/ユニットの所有関係**
- コース・ユニットは会員個人が主催/投稿する(団体の所有物ではない)
- 団体所属の有無に関わらず、会員はコース・ユニットを作成できる
- 団体は、所属メンバーのコース/ユニットを集約・可視化するレイヤー

**ビューの二重性**
| ビュー | スコープ | 主な用途 |
|---|---|---|
| 個人ビュー(マイ〇〇) | 自分が主催/投稿したもの | 本人による作成・編集・削除 |
| 団体集約ビュー(団体〇〇) | 自団体メンバー全員分 | 団体管理者による集約確認と介入 |

**編集権限**
- 主催者/投稿者本人: フル編集可
- 団体管理者(他人のコース/ユニットに対して): 編集・削除可(上位権限)
- その他会員: アクセス不可

## アクセス導線

```
[ちょい英語ログイン(既存)]
  ↓
URLを直接入力してアクセス
  ↓
[トピックメーカー専用暫定プロフィール画面]
  ├─ 「所属団体: 〇〇」表示(所属時)
  │   └─ 管理者の場合のみリンク → [A-T2 団体ダッシュボード]
  ├─ 「団体を新規登録」ボタン(無所属時)     → [A-T1 団体新規登録]
  ├─ 「マイコース」リンク                   → [A-MC1 マイコース一覧]
  └─ 「マイユニット」リンク                 → [A-MU1 マイユニット一覧]
```

## ナビゲーション構造(管理コンソール左ペイン)

会員のロールごとに表示が変わる。ドッグフーディングの観点で、
全ロール向けのパターンを実装する。

### 無所属会員

```
管理コンソール
▼ マイコンテンツ
  マイコース            → [A-MC1]
  マイユニット          → [A-MU1]
▼ 団体
  団体を新規登録         → [A-T1]
```

### 団体所属(一般)会員

```
管理コンソール
▼ マイコンテンツ
  マイコース            → [A-MC1]
  マイユニット          → [A-MU1]
▼ 所属団体
  〇〇株式会社         ← 名称表示のみ(リンクでない)
```

### 団体管理者

```
管理コンソール
▼ マイコンテンツ
  マイコース            → [A-MC1]
  マイユニット          → [A-MU1]
▼ 自団体(〇〇株式会社)
  団体ダッシュボード    → [A-T2]
  会員                  → [A-M1]
  団体コース            → [A-C1]
  団体ユニット          → [A-U1]
```

## 必要なユースケース

### 既存

- UC-01: 団体を登録する
- UC-02: 団体情報を編集する
- UC-03: 会員を招待・追加する
- UC-04: 会員のロールを変更する
- UC-05: 会員を削除する
- UC-10〜17: コース管理系
- UC-20〜26: ユニット管理系

### 新規追加

- **UC-18**: 自団体メンバーが主催するコースを一覧・確認する(団体管理者)
- **UC-19**: 自団体メンバーが主催するコースを編集・削除する(団体管理者)
- **UC-27**: 自団体メンバーが投稿したユニットを一覧・確認する(団体管理者)
- **UC-28**: 自団体メンバーが投稿したユニットを編集・削除する(団体管理者)

### システム運用 (対象外)

- UC-06: isトピックメーカー属性の設定

## フェーズ1特有の扱い(暫定プロフィール画面のみ)

- プロフィール画面自体はトピックメーカー社員のみが知るURLで提供
- 社員が以下のロールを**自由に切り替えてドッグフーディング**できるよう、
  ユーザー側のロール変更手段を別途用意(運用DBで切り替え or 専用ツール)
  - 無所属状態
  - 団体一般会員状態
  - 団体管理者状態
- ドッグフーディング終了後、フェーズ2ではプロフィール画面を正規化する
